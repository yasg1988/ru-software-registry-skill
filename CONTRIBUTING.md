# Правила участия

Спасибо за интерес к проекту.

Этот репозиторий содержит skill для Codex, который помогает готовить проекты к подаче в реестр российского ПО Минцифры.

## Как предложить изменение

1. Создайте issue с описанием проблемы или предложения.
2. Для кода и документации создайте pull request в ветку `main`.
3. Опишите, какую задачу решает изменение.
4. Приложите результат проверки, если изменение затрагивает скрипты или workflow.

## Проверки перед pull request

Запустите:

```powershell
npm run validate
npm run audit
```

Если менялись npm-метаданные, дополнительно проверьте состав пакета:

```powershell
npm pack --dry-run
```

## Требования к изменениям

- Не добавляйте реальные токены, пароли, ключи, `.env` и персональные данные.
- Не коммитьте `data/applicant.local.json` и папки `registry_ru/`.
- Пишите документацию на русском языке, если она предназначена для пользователей skill.
- Сохраняйте совместимость с Windows PowerShell, потому что основной сценарий использования рассчитан на локальную работу в Codex на Windows.
- Не выполняйте юридически значимые действия в автоматизации без явного подтверждения пользователя.

## Версии и релизы

Версия хранится в `package.json`.

Для релиза используется схема:

```powershell
npm version <version> --no-git-tag-version
git commit -am "Release <version>"
git tag v<version>
git push origin main
git push origin v<version>
```

Публикация npm выполняется через GitHub Actions workflow `Publish npm`.

Синхронизация с GitVerse выполняется через workflow `Sync GitVerse`.
