# Политика безопасности

## Поддерживаемые версии

Поддерживается актуальная опубликованная версия пакета `@yasg1988/ru-software-registry-skill` и текущая ветка `main`.

## Как сообщить об уязвимости

Если вы нашли уязвимость, не публикуйте ее в открытых issue до первичной проверки.

Напишите автору через GitHub Security Advisory или создайте приватное обращение доступным способом связи, указанным в профиле репозитория.

В сообщении укажите:

- версию пакета;
- операционную систему и окружение;
- шаги воспроизведения;
- возможное влияние;
- фрагменты логов без токенов, паролей и персональных данных.

## Сроки реакции

Ориентировочный срок первичного ответа: 1-3 рабочих дня.

## Секреты и персональные данные

Не прикладывайте к issue, pull request или логам:

- npm/GitHub/GitVerse/Docker токены;
- приватные SSH-ключи;
- файлы `.env`;
- реальные персональные данные заявителей или пользователей;
- документы с паспортными данными, адресами или подписями.

Для локальных данных правообладателя используйте файл `data/applicant.local.json`; он добавлен в `.gitignore` и не должен попадать в публичный репозиторий.
